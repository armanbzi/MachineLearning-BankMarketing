# Bank Marketing — Term Deposit Subscription Prediction

**ML Lab Project — UPC-FIB, MDS 2026**
**Author:** Arman Bazarchi

A full machine-learning pipeline on the [UCI Bank Marketing dataset](https://archive.ics.uci.edu/dataset/222/bank+marketing),
predicting whether a client subscribes to a term deposit (binary classification). The project runs
across five phases — from unsupervised structure to a validated final model.

---

## Dataset

- **Source:** UCI Bank Marketing (`bank-additional-full.csv`), downloaded automatically by
  `preprocessing.ipynb`.
- **Size:** 41,188 rows × 20 features + binary target `y`.
- **Class imbalance:** 88.7% "no" / 11.3% "yes" (≈ 7.9 : 1).
- **Key modelling decisions:**
  - `duration` is **excluded** from all supervised models — it is recorded *after* the call, so using
    it would be target leakage. (It is the single strongest predictor, which is why its removal sets a
    realistic performance ceiling.)
  - `pdays` is **dropped** — 96.3% sentinel (999) values; its signal is already covered by `poutcome`
    and `previous`.

## Repository structure

| File | Phase | Description |
|------|-------|-------------|
| `preprocessing.ipynb` | — | Loads data, EDA, encoding, scaling. **Run this first** — it writes the `data/` artifacts every other notebook reads. |
| `phase1_unsupervised_analysis.ipynb` | I | PCA + clustering (K-Means, GMM); structural analysis. |
| `phase2_linear_regularized_models.ipynb` | II | Logistic Regression; Ridge/Lasso (Linear Probability Model); effect of regularization (λ). |
| `phase3_kernel_svm.ipynb` | III | SVM with linear / polynomial / RBF kernels + grid search. |

| `phase5_model_selection.ipynb` | V | Nested cross-validation, paired t-test, final model selection. |
| `dataset_discovery.ipynb` | — | Initial exploratory scratch notebook (not part of the pipeline). |
| `requirements.txt` | — | Pinned dependencies. |
| `data/` | — | Auto-generated by `preprocessing.ipynb` (git-ignored — not in the repo). |

## Setup

Requires **Python 3.13** (tested on 3.13.9).

```bash
# (recommended) create an isolated environment
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

# install pinned dependencies
pip install -r requirements.txt
```

> **Note:** `shap` (used for TreeSHAP in Phase IV) is included in `requirements.txt` — it is an extra
> dependency beyond the usual scientific stack.

---

## How to run

The notebooks must be run **in order**, because every phase reads the processed data produced by
preprocessing:

1. **`preprocessing.ipynb`** — run it top-to-bottom first. It downloads the dataset from UCI and
   writes the processed artifacts into `data/` (`df_raw.csv`, `df_processed.csv`, `X_scaled.csv`,
   `cat_features.csv`, `y.csv`).
2. **`phase1` → `phase5`** — run any/all of them afterwards, in any order (each only depends on
   `data/`, not on the other phases).

```bash
jupyter lab        # or: jupyter notebook
```

To run a notebook non-interactively (e.g. to regenerate all outputs):

```bash
jupyter nbconvert --to notebook --execute --inplace preprocessing.ipynb
jupyter nbconvert --to notebook --execute --inplace phase4_ensemble_methods.ipynb
```

> An internet connection is required the first time, for `preprocessing.ipynb` to download the
> dataset from the UCI archive.

## Reproducibility

- **Deterministic.** Every notebook sets `np.random.seed(42)` and every stochastic step
  (`train_test_split`, all models, CV folds, `permutation_importance`, sampling) uses
  `random_state=42`. With the pinned versions in `requirements.txt`, results reproduce exactly.
- **Version-sensitive.** Defaults differ across scikit-learn releases, so use the pinned versions —
  in particular `scikit-learn==1.8.0`. A different version may shift the reported numbers slightly.
- **Data is not committed** (`data/` is git-ignored); it is regenerated by `preprocessing.ipynb`.
